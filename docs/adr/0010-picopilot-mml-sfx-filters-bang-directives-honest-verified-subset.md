# picopilot-MML exposes PICO-8's SFX filters as `!`-directives, and ships only the byte-verified filter subset (REVERB is a single switch, not 2 levels)

picopilot-MML expresses PICO-8's 5 per-SFX filters as `!`-prefixed SFX-level directives (`!noiz !buzz !detune1/!detune2 !reverb !dampen[2]`) that set the first `__sfx__` header byte, closing the "designed sounds" gap (explosion = DAMPEN body + REVERB tail) the audio dogfood surfaced. Two decisions carry real trade-offs and are recorded here: the `!`-directive syntax (over a `--filter` flag or bare mnemonics), and shipping REVERB as a single on/off switch rather than the manual's 2 levels, because the decoded byte (finding A.7) has no non-colliding bit for a REVERB level-2 under the verified layout.

## Context

The audio spike (ADR-0005) deferred filters; the audio-example dogfood then proved they are a TOOL gap, not a skill gap (a maxed-out `@6` boom with a hand-drawn pitch+volume envelope still reads as a machine-gun rattle, because "boom body" = DAMPEN and "tail" = REVERB, which no per-note pitch/waveform/volume MML can synthesize). The `audio-mml-sfx-filters` task decoded the filter byte layout byte-for-byte on PICO-8 v0.2.7 (finding A.7): the 5 filters live in the FIRST header byte of the `__sfx__` row (an 8-bit bitfield, verified to round-trip verbatim), with bit1=NOIZ, bit2=BUZZ, bit3=DETUNE-1, bit4=DETUNE-2, bit5=REVERB, bits6-7=DAMPEN (2 levels), bit0=cosmetic editor mode.

## Decisions

- **Filter syntax = `!`-prefixed SFX-level directives** (`!dampen`, `!dampen2`, `!reverb`, `!noiz`, `!buzz`, `!detune1`, `!detune2`), placeable anywhere in the string like `s<N>` speed. A `!` head is UNUSED in the note grammar, so it cannot silently re-mean a note-letter sequence — the exact mnemonic-collision trap that killed the effect mnemonics `sl vb dr` (ADR-0008: those were bare note-plus-length sequences). DAMPEN takes an optional level 1..2 (`!dampen`=1, `!dampen2`=2); an out-of-range level (`!dampen3`, `!reverb2`, `!noiz2`, `!detune` without 1/2) is a structured `audio-mml-filter-level-out-of-range` refusal, never a silent clamp (consistent with ADR-0008's pitch/overflow stance).

- **REVERB is exposed as a single on/off switch, not 2 levels.** The manual lists both DAMPEN ("2 different levels") and REVERB ("delay of 2 or 4 ticks") as 2-level, but the 8-bit filter byte has room for only one of them to be a 2-bit field: after mode(bit0) + NOIZ + BUZZ + DETUNE-1/2 (bits 1-4) + REVERB-on (bit5) + DAMPEN (bits 6-7), all 8 bits are spent, and a REVERB level-2 bit would collide with DAMPEN's level-2 bit (bit 6). DAMPEN's 2 levels were DIRECTLY verified (0x80 vs 0xc0 = progressively stronger low-pass); REVERB's second level could not be isolated (the box's real-time A/V capture went durably unavailable mid-spike) and has no non-colliding home. So picopilot ships the HONEST, byte-verified, collision-free subset: DAMPEN 2 levels, REVERB single switch. Because PICO-8 stores the byte verbatim, adding a REVERB level-2 later (if a future spike isolates its bit) is a one-constant change.

## Considered options

- **A `--filter noiz,dampen=2,reverb` command flag (rejected as the primary surface).** The filters are per-SFX METADATA, so they belong in the MML string next to the other SFX-level directives (`s<N>` speed, `{`/`}` loop), not split across a separate flag surface where they would drift from the source they modify. The `!`-directive keeps one authored artifact.
- **Bare-mnemonic filter tokens (rejected).** Any un-prefixed keyword risks colliding with the tracker-row note grammar (ADR-0008's lesson); the `!` prefix makes filters unambiguously not-notes.
- **Claiming a REVERB level-2 with a guessed bit (rejected).** Shipping an unverified, likely-colliding level-2 would produce wrong-sounding output the author cannot see the cause of, the exact silent-wrongness picopilot refuses elsewhere. Better to expose the verified subset and record the open question in the finding.
